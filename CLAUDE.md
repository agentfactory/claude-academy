# AI Marketing Officer — Project Documentation

## Project Summary

An autonomous AI Marketing Officer system built with Claude Code skills, hooks, and sub-agents. Takes any input (topic, YouTube URL, TikTok URL, article, PDF, audio, image) and generates platform-optimized social media content with visuals, then publishes/schedules across Twitter, LinkedIn, Instagram, and Facebook via the Blotato API.

## API Integrations

### Blotato API
- **Base URL**: `https://backend.blotato.com/v2`
- **Authentication**: Header `blotato-api-key` with API key value
- **API Key Location**: `.claude/settings.local.json` → `env.BLOTATO_API_KEY`
- **API Dashboard**: `https://my.blotato.com/api-dashboard` (debug requests here)

### Key Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/v2/users/me/accounts` | GET | List connected social accounts |
| `/v2/users/me/accounts/:id/subaccounts` | GET | Get Facebook/LinkedIn pageId |
| `/v2/posts` | POST | Publish/schedule posts (30 req/min) |
| `/v2/posts/:id` | GET | Poll post status (60 req/min) |
| `/v2/source-resolutions-v3` | POST | Extract content from URLs (30 req/min) |
| `/v2/source-resolutions-v3/:id` | GET | Poll extraction status |
| `/v2/videos/from-templates` | POST | Create visuals (1 req/min) |
| `/v2/videos/creations/:id` | GET | Poll visual status |
| `/v2/videos/templates` | GET | List available templates |

## Skills Directory

### `/post` — Single Post Creator & Publisher
- **Location**: `.claude/skills/post/SKILL.md`
- **Usage**: `/post [topic or URL] [platform or "all"]`
- **What it does**: Detects input type, extracts content, generates 2-3 draft options, creates visuals, publishes/schedules to chosen platform(s)
- **Multi-platform**: Use `all` as platform to spawn sub-agents for parallel publishing

### `/plan-week` — Weekly Content Planner
- **Location**: `.claude/skills/plan-week/SKILL.md`
- **Usage**: `/plan-week [topic or URL]`
- **What it does**: Extracts source content, generates 10 angles, creates 20 posts (5 days x 4 platforms), generates 5 infographics, saves plan to `content-plan.md`, waits for approval, then batch-schedules via sub-agents

## Brand Voice

Brand voice guidelines are embedded directly in `.claude/skills/post/SKILL.md` under the "BRAND VOICE" section.

### Platform Voices
- **Twitter**: Alex Hormozi style — punchy, short-form, direct, pattern-interrupt hooks
- **LinkedIn**: Justin Welsh style — story-driven, structured, outcome-focused
- **Instagram**: Visual-first captions, hooks, moderate emoji, 3-5 hashtags
- **Facebook**: Conversational, community-oriented, engagement questions

### Humanize AI Writing Rules
- No em dashes, no AI filler phrases, no buzzwords
- Short punchy sentences mixed with longer ones
- Concrete examples, specific numbers, active voice
- Natural contractions, platform-native formatting

## Quality Gates

### Pre-Publish Validation Hook
- **Location**: `.claude/hooks/validate-post.sh`
- **Registered in**: `.claude/settings.json` under `hooks.PreToolUse`
- **Triggers on**: Any bash command containing `curl` to `backend.blotato.com/v2/posts`

### Checks Performed
1. **Em dash check** — blocks posts containing `—`
2. **Character limit check** — Twitter: 280, LinkedIn: 3000, Instagram: 2200, Facebook: 63206
3. **Banned words check** — rejects: guaranteed, hack, hustle, grind, guru, ninja, rockstar, synergy, disrupt, leverage, circle back, deep dive, move the needle, game-changer, thought leader
4. **Required media check** — Instagram posts must have at least one mediaUrl
5. **LLM virality score** — on pass, outputs the hook/opening line and asks Claude to score it 8/10+ before publishing; if below threshold, rewrite the hook

### Updating Quality Gates
- Edit banned words list in `.claude/hooks/validate-post.sh` (the `BANNED_WORDS` array)
- Character limits are hardcoded per platform in the same script
- Hook exits with code 2 to block, code 0 to allow

## Scheduling Configuration

- **Posting frequency**: 1 post per platform per day, weekdays only (Mon-Fri)
- **Platforms**: Twitter, LinkedIn, Instagram, Facebook
- **Default scheduling**: `useNextFreeSlot: true` (fills Blotato calendar slots in order)
- **Calendar slots**: Must be configured in Blotato: Calendar > Weekly Schedule > Add Slot
- **Safety**: Never auto-publish during development; always confirm before posting

## Conventions

- **Default visual type**: Whiteboard infographic (not carousel)
- **Drafts**: Always show 2-3 options before publishing
- **Content plan file**: `content-plan.md` (generated by `/plan-week`)
- **Post log**: `post-log.md` (append-only log of all published posts)
- **API key storage**: `.claude/settings.local.json` (gitignored)
- **Sub-agents**: Used for multi-platform parallel publishing

## Known Issues / Future Improvements

- Visual creation rate limit is 1/min — batch visual generation for `/plan-week` takes ~5 minutes
- Blotato help site blocks automated fetches (403) — use web search for doc lookups
- Instagram requires media — the hook enforces this but ensure visuals are generated first
- Future: Add Instagram Reels/Stories support, Pinterest, Bluesky, Threads, YouTube
- Future: Add local image/video folder scanning for carousel posts
- Future: Add MCP integration with n8n/Make.com for advanced orchestration
- Future: Add analytics tracking (engagement rates, best performing posts)
